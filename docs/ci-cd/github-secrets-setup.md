# GitHub Secrets & Variables Setup

Checklist for configuring the CI/CD pipeline (Azure deployment branch).

Go to: **Repo → Settings → Secrets and variables → Actions**

## Repository Variables (Settings → Variables → New repository variable)

| Variable | Value | Used by | Example |
|----------|-------|---------|---------|
| `ACR_LOGIN_SERVER` | Azure Container Registry login server | CI (docker build/push), CD (image refs) | `<your-acr-name>.azurecr.io` |

## Repository Secrets (Settings → Secrets → New repository secret)

| Secret | What it is | Used by | How to get it |
|--------|-----------|---------|---------------|
| `ACR_USERNAME` | ACR admin username or service principal client ID | CI (docker login), CD (VM docker login) | `az acr credential show --name <your-acr-name> --query username -o tsv` |
| `ACR_PASSWORD` | ACR admin password or service principal secret | CI (docker login), CD (VM docker login) | `az acr credential show --name <your-acr-name> --query "passwords[0].value" -o tsv` |
| `AZURE_CREDENTIALS` | Service principal JSON for `azure/login` action | CD (get VM IP, smoke tests) | See "Create Service Principal" below |
| `AZURE_VM_SSH_KEY` | Private SSH key for the deploy VM | CD (SSH deploy) | `cat ~/.ssh/id_rsa` (the key used during `az vm create --generate-ssh-keys`) |
| `AZURE_VM_HOST_KEY` | VM SSH host public key (prevents MITM) | CD (known_hosts) | `ssh-keyscan <VM_IP>` (run once after VM provisioning) |

## GitHub Environment (Settings → Environments → New environment)

| Environment | Purpose | Recommended settings |
|-------------|---------|---------------------|
| `dev` | Approval gate before Azure deployment | Add yourself as required reviewer; no branch restriction needed for MVP |

## How to Create Each Secret

### ACR credentials (admin mode)

```bash
# Enable admin access on the existing ACR
az acr update --name <your-acr-name> --admin-enabled true

# Get credentials
az acr credential show --name <your-acr-name>
# → username = <your-acr-name>
# → passwords[0].value = <paste as ACR_PASSWORD>
```

### Azure service principal (for azure/login action)

```bash
az ad sp create-for-rbac \
  --name "github-api-observatory" \
  --role Contributor \
  --scopes /subscriptions/<SUBSCRIPTION_ID>/resourceGroups/api-observatory-rg \
  --sdk-auth
```

Copy the entire JSON output as `AZURE_CREDENTIALS`.

### VM SSH host key

```bash
# After VM is provisioned, run once:
ssh-keyscan <VM_IP>
# Copy the output (all lines) as AZURE_VM_HOST_KEY
```

### VM SSH private key

```bash
# The key generated by az vm create --generate-ssh-keys
cat ~/.ssh/id_rsa
# Copy the entire key (including BEGIN/END lines) as AZURE_VM_SSH_KEY
```

## Verification Checklist

After adding all secrets and variables:

- [ ] `ACR_LOGIN_SERVER` variable is set (not a secret — it's referenced in image tags visible in logs)
- [ ] `ACR_USERNAME` and `ACR_PASSWORD` secrets are set
- [ ] `AZURE_CREDENTIALS` secret contains valid service principal JSON
- [ ] `AZURE_VM_SSH_KEY` secret contains the private key (with newlines preserved)
- [ ] `AZURE_VM_HOST_KEY` secret contains the host key from `ssh-keyscan`
- [ ] `dev` environment exists with at least one required reviewer
- [ ] ACR admin access is enabled: `az acr update --name <your-acr-name> --admin-enabled true`
- [ ] Test ACR login locally: `docker login <your-acr-name>.azurecr.io`
